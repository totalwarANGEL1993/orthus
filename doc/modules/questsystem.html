Module questsystem

This is an approach to create a RoaE like quest system.

Quests can be either successfully finished or failed. If a quest is finished successfully the rewards will be executed. If the quest fails the reprisals will be executed instead. Quests are controlled by behaviors. Behind a behavior is predefined code that is executed with certain parameters. It is also possible to create a custom behavior for each type.

Types are:

  • Condition: Triggering the quest if conditions are true
  • Objective: What the player has to do to win (or to fail)
  • Reward: Rewards for successfully finishing a quest.
  • Reprisal: Pubishments after the quest failed.
But reprisals and rewards are both callbacks!

A quest is generated like this:

local QuestID = new(QuestTemplate, "SomeName", SomeObjectives, SomeConditions, 1, 0, SomeRewards, SomeReprisals)

Some of the behavior might be redefined in the qsb.questbehavior abstraction layer.

Required modules:

  • qsb.oop

Constants

Callbacks
Actions that are performed when a quest is finished.

Fields:

  • MapScriptFunction Calls a user function as reward. 
    {Callbacks.MapScriptFunction, _Function, _ArgumentList...}
  • Defeat Player looses the game. 
    {Callbacks.Defeat}
  • Victory Player wins the game. 
    {Callbacks.Victory}
  • Briefing Calls a function with a briefing. The function is expected to return the briefing id! Attach only one briefing to one quest! 
    {Callbacks.Briefing, _Briefing}
  • ChangePlayer Changes the owner of the entity 
    {Callbacks.ChangePlayer, _Entity, _Owner}
  • Message Displays a simple message on screen. 
    {Callbacks.Message, _Message}
  • DestroyEntity Replace a named entity or millitary group with a script entity. 
    {Callbacks.Message, _ScriptName}
  • DestroyEffect Destroy a named graphic effect. 
    {Callbacks.Message, _EffectName}
  • CreateEntity Replaces a script entity with a new entity. The new entity will have the same owner and orientation as the script entity. 
    {Callbacks.CreateEntity, _ScriptName, _Type}
  • CreateGroup Replaces a script entity with a military group. The group will have the same owner and orientation as the script entity. 
    {Callbacks.CreateGroup, _ScriptName, _Type, _Soldiers}
  • CreateEffect Creates an effect at the position. 
    {Callbacks.DestroyEffect, _EffectName, _EffectType, _Position}
  • Diplomacy Changes the diplomacy state between two players. 
    {Callbacks.Diplomacy, _PlayerID1, _PlayerID2, _State}
  • Resource Give or remove resources from the player. 
    {Callbacks.Resource, _ResourceType, _Amount}
  • RemoveQuest Removes a quest from the quest book. The quest itself stays untouched. 
    {Callbacks.RemoveQuest, _QuestName}
  • QuestSucceed Let a active quest succeed. 
    {Callbacks.QuestSucceed, _QuestName}
  • QuestFail Let a active quest fail. 
    {Callbacks.QuestFail, _QuestName}
  • QuestInterrupt Interrupts a quest even when it was not startet. 
    {Callbacks.QuestInterrupt, _QuestName}
  • QuestActivate Activates a quest when it was not triggered 
    {Callbacks.QuestActivate, _QuestName}
  • QuestRestart Restarts a quest so that it can be triggered again. 
    {Callbacks.QuestRestart, _QuestName}
  • Technology Change the state of a technology. Possible technology states:
    • Allow: A technology will be allowed
    • Research: A technology is set as research
    • Forbid: A technology is unaccessable
    {Callbacks.Technology, _Tech, _State}
  • CreateMarker Creates an minimap marker or minimap pulsar at the position. 
    {Callbacks.CreateMarker, _Type, _PositionTable}
  • DestroyMarker Removes a minimap marker or pulsar at the position. 
    {Callbacks.DestroyMarker, _PositionTable}
  • RevealArea Explores an area around a script entity. 
    {Callbacks.RevealArea, _AreaCenter, _Explore}
  • ConcealArea Removes the exploration of an area. 
    {Callbacks.ConcealArea, _AreaCenter}
  • Move Removes the exploration of an area. 
    {Callbacks.Move, _Entity, _Destination}
Conditions
Condition types that triggers quests.

Fields:

  • NeverTriggered Quest will never be triggered. 
    {Conditions.NeverTriggered}
  • Time Quest will be triggered after some time after mapstart. 
    {Conditions.Time, _Waiitime}
  • MapScriptFunction Quest will be triggered when a user function returns true. 
    {Conditions.MapScriptFunction, _Function, _ArgumentList...}
  • Diplomacy Starts the quest when a diplomatic state is reached. 
    {Conditions.Diplomacy, _Player, _Diplomacy}
  • Briefing Starts the quest, after a briefing of another is finished. 
    {Conditions.Briefing, _QuestName}
  • QuestSuccess Starts a quest when another quest is finished successfully. 
    {Conditions.QuestSuccess, _QuestName}
  • QuestFailure Starts a quest when another quest has failed. 
    {Conditions.QuestFailure, _QuestName}
  • QuestInterrupt Starts a quest when another quest is interrupted. 
    {Conditions.QuestInterrupt, _QuestName}
  • QuestActive Starts a quest when another quest is active. 
    {Conditions.QuestActive, _QuestName}
  • QuestOver Starts a quest when another quest either failed or is finished sucessfully. 
    {Conditions.QuestOver, _QuestName}
  • QuestNotTriggered Starts a quest when another quest has not yet been triggered. 
    {Conditions.QuestNotTriggered, _QuestName}
  • Payday Starts the quest on the next payday 
    {Conditions.Payday}
  • EntityDestroyed Starts the quest when a entity does not exist. 
    {Conditions.EntityDestroyed, _ScriptName}
  • WeatherState Starts the quest when the weather changed to a weather state. 
    {Conditions.WeatherState, _State}
  • QuestAndQuest Starts the quest when both quest have the same result. 
    {Conditions.QuestAndQuest, _Quest1, _Quest2, _Result}
  • QuestOrQuest Starts the quest when one or both quest have the same result. 
    {Conditions.QuestOrQuest, _Quest1, _Quest2, _Result}
  • QuestXorQuest Starts the quest when one or the other quest but NOT both have the same result. 
    {Conditions.QuestXorQuest, _Quest1, _Quest2, _Result}

Objectives
Objective types the player musst fulfill to succeed.

Fields:

  • MapScriptFunction Quest result will be decided by a user function 
    {Objectives.MapScriptFunction, _Function, _ArgumentList...}
  • InstantFailure Quest will allways instantly fail. 
    {Objectives.InstantFailure}
  • InstantSuccess Quest will allways instantly succeed. 
    {Objectives.InstantSuccess}
  • NoChange Quest never changes 
    {Objectives.NoChange}
  • Destroy Destroy a unit, a whole army or make a hero loose consciousness. 
    {Objectives.Destroy, _target}
  • Create Create units or buildings in area. 
    {Objectives.Create, _entityType, _position, _range, _amount, _mark, _changeOwner}
  • Diplomacy Reach a diplomatic state to another player. 
    {Objectives.Diplomacy, _OtherPlayer, _State}
  • Produce Gain a amount of resources. 
    {Objectives.Produce, _Resource, _Amount, _WithoutRaw}
  • Protect Shield a entity from any harm. 
    {Objectives.Protect, _Target}
  • EntityDistance A entity must be near to a position on the map. 
    {Objectives.EntityDistance, _Entity, _Target, _Distance, _LowerThan}
  • Workers Reach a number of workers in the settlement. 
    {Objectives.Workers, _Amount, _LowerThan, _OtherPlayer}
  • Motivation Reach a minimum amount of average motivation for the workers. 
    {Objectives.Motivation, _Amount, _LowerThan, _OtherPlayer}
  • Units Create an amount of units. 
    {Objectives.Units, _Type, _Amount}
  • Technology Research a technology 
    {Objectives.Technology, _Tech}
  • Headquarter Upgrade the headquarters one or two times 
    {Objectives.Headquarters, _Upgrades}
  • DestroyType Destroy an amount of entities of type. 
    {Objectives.DestroyType, _PlayerID, _Type, _Amount}
  • DestroyCategory Destroy an amount of entities in a category. 
    {Objectives.DestroyCategory, _PlayerID, _Category, _Amount}
  • Tribute The player must pay a tribute to succeed 
    {Objectives.Tribute, _CostsTable, _Message}
  • Settlers Reach an overall amount of settlers in the settlement. 
    {Objectives.Settlers, _Amount, _LowerThan, _OtherPlayer}
  • Soldiers Reach a number of military units in the settlement. 
    {Objectives.Soldiers, _Amount, _LowerThan, _OtherPlayer}
  • WeatherState The player must change the weather to the weather state. 
    {Objectives.WeatherState, _State}
  • DestroyAllPlayerUnits The player must destroy all buildings and units of the player. 
    {Objectives.DestroyAllPlayerUnits, _PlayerID}
  • Quest The player must complete the quest with the desired result. If the quest is not required, failing the result will not fail the quest. 
    {Objectives.Quest, _Quest, _Result, _Required}
  • Bridge The player must build a bridge in the marked area. Because bridges loose their script names often, use a XD_ScriptEntity instead of the site. 
    {Objectives.Bridge, _AreaCenter, _AreaSize}
