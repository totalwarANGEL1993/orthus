Module ext.troopgenerator

This module allows to create AI player and armies.

Altough this module is using parts of the AI functions for handling normal armies this script is not bound to the limit of 8 armies per player or 8 troops per army.

Armies have a strict behavior defined by their FSM. They will attack certain positions or patrol between them. Sub behaviors like how the army reacts to attacks or which troops are spawned can be configured.

If you are an beginner you should stick to the options the quest behaviors give you. They are enough in most of the cases.

Required modules:

  • qsb.oop
  • qsb.core.questsync
  • qsb.core.questtools

ChangeArmyBehavior (_PlayerID, _ArmyID, _Behavior, _Controller)
Overrides the behavior of the army with the given function.

Note: Only change behavior if you know what you are doing! You might break the army controller!

Behavior Description
MemberIsAttacked Called when ever an enemy attacks an member of the army.
SelectPurchasedType Note: Only used if the army purchases troops!
Passes the catalog to the controller. The controller must set the catalog iterator. Use this to change how an army recruits their members. You can prioritize certain types.
SelectSpawnedType Note: Only used if the army spawns troops!
Passes the catalog to the controller. The controller must set the catalog iterator.
SelectAttackTarget Passes the list of reachable attack targets to the controller function. The controller must select one position, set it as path and as target.
SelectPatrolTarget Passes the list of reachable patrol targets to the controller function. The controller must set the path and the first waypoint. Army will loop over the waypoints starting from the selected one.

Parameters:

  • _PlayerID number ID of player
  • _ArmyID number ID of army
  • _Behavior number ID of behavior
  • _Controller number Controller function

Usage:

    ChangeArmyBehavior(2, FrontalAttackArmyID, ArmyBehaviors.SelectPurchasedType, CavalryOnlyRecruiter);
CreateAIPlayer (_PlayerID, _TechLevel, _SerfAmount, _HomePosition, _Strength, _Construct, _Rebuild)
Creates an AI player and sets the technology level.

Use this function or the behavior to initalize the AI. An AI must first be initalized before an army can be created.

Note: If you decide to allow to recruit armies keep in mind that they will have 12 groups instead of 8!

Parameters:

  • _PlayerID number PlayerID
  • _TechLevel number Technology level [1|4]
  • _SerfAmount number (optional) Amount of serfs
  • _HomePosition string (optional) Default army home position
  • _Strength number (optional) Armies to recruit (0 = off)
  • _Construct boolean (optional) AI can construct buildings
  • _Rebuild boolean (optional) AI rebuilds (construction required)

Usage:

    CreateAIPlayer(2, 4);
CreateAIPlayerArmy (_ArmyName, _PlayerID, _Strength, _Position, _Area, _TroopTypes)
Initalizes an army that is recruited by the AI player. Armies can also be created with the behavior interface. This is a simple type of army that can be configured by placing and naming script entities. The army name must be unique for the player!

The AI player must be initalized first!

Use script entities named with PlayerX_AttackTargetY to define positions that will be attacked by the army. Replace X with the player ID and Y with a unique number starting by 1.

Also you can use entities named with PlayerX_PatrolPointY to define positions were the army will patrol. Also replace X with the player ID and Y with a unique number starting by 1.

Parameters:

  • _ArmyName string Army identifier
  • _PlayerID number Owner of army
  • _Strength number Strength of army [1|8]
  • _Position string Home Position of army
  • _Area number Action range of the army
  • _TroopTypes table Upgrade categories to recruit

Returns:

    number Army ID

Usage:

    CreateAIPlayerArmy("Foo", 2, 8, "armyPos1", 5000, TroopTypeTable);

CreateAIPlayerSpawnArmy (_ArmyName, _PlayerID, _Strength, _Position, _Spawner, _Area, _Respawn, ...)
Initalizes an army that is spawned until a generator entity is destroyed. Armies can also be created with the behavior interface. This is a simple type of army that can be configured by placing and naming script entities. The army name must be unique for the player!

The AI player must be initalized first!

Define a list of troops that are sparned in this order. Troops will be spawrned as long as the generator exists.

Use script entities named with PlayerX_AttackTargetY to define positions that will be attacked by the army. Replace X with the player ID and Y with a unique number starting by 1.

Also you can use entities named with PlayerX_PatrolPointY to define positions were the army will patrol. Also replace X with the player ID and Y with a unique number starting by 1.

Parameters:

  • _ArmyName string Army identifier
  • _PlayerID number Owner of army.
  • _Strength number Strength of army [1|8]
  • _Position string Home Position of army
  • _Spawner string Name of generator
  • _Area number Action range of the army
  • _Respawn number Time till troops are refreshed
  • ... List of types to spawn

Usage:

    CreateAIPlayerSpawnArmy(
    "Bar", 2, 8, "armyPos1", "lifethread", 5000,
    {Entities.PU_LeaderSword2, 3},
    {Entities.PU_LeaderBow2, 3},
    {Entities.PV_Cannon2, 0}
);

Fields:

  • City City troop types
  • BlackKnight Black knight troop types
  • Bandit Bandit troop types
  • Barbarian Barbarian troop types
  • Evil Evil troop types
ArmyStates
Table of army sub behaviors.

Fields:

  • Decide Army is in default state and selects next action
  • Attack Army walks to the destination and attacks enemies
  • Guard Army patrols over their path
  • Retreat Army is retreating to the home base
  • Refill Army is recruiting or respawning until full
GroupTargetingPriorities
List of targeting priorities for the different categories of enemies. The higher the number the higher the priority. Negative values are setting the priority below the average. This can be used to ignore enemy categories if better targets are available.

Note: Use EntityCategories here!

Fields:

  • Cannon Attack priority for cannons
  • HeavyCavalry Attack priority for heavy cavalry
  • Sword Attack priority for swordsmen
  • Spear Attack priority for spearmen
  • Ranged Attack priority for bowmen and light cavalry
  • Rifle Attack priority for marksmen
