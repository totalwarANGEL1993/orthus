Module questbehavior

This is an abstraction layer of the quest system, allowing the user to create quests much more simple.

Also some behavior are recoded and there is a better AI control script for armies than the default controller.

Required modules:

  • qsb.lib.oop
  • qsb.lib.questtools
  • qsb.core.questsystem
  • qsb.core.questdebug
  • qsb.ext.interaction
  • qsb.ext.information
  • qsb.ext.onscreeninfo
  • qsb.ext.troopgenerator

CreateQuest (_Data)
Creates an quest from the given table. If the table contains a description, the quest will insert itself into the questbook when it is triggered and erase it, after it is finished or interrupted.

Keep in mind that there can be only 8 entries in the quest book unless you use extra 3 or the standalone version of mcb's quest!

Table contains of the following entries:

  • Name: Name of quest
  • Receiver: Player that receives the quest
  • Time: Time, until the quest is automatically over
  • Description: Quest information displayed in the quest book. (Mostly identical to Logic.CreateQuest or Logic.CreateQuestEx)
After the fields the behavior constructors are called.

Parameters:

  • _Data table Quest table

Returns:

  1. number Quest id
  2. table Quest instance

Usage:

    CreateQuest {
    Name = "VictoryCondition",
    Description = {
        Title = "Justice!",
        Text  = "Time for paybeck. Destroy your enemy!",
        Type  = MAINQUEST_OPEN,
        Info  = 1
    },

    Goal_DestroyPlayer(2, "HQ2"),
    Reward_Victory(),
    Trigger_Time(0)
}

Reward_AI_CreateArmy (_ArmyName, _PlayerID, _Strength, _Position, _Area, _TroopType)
Defines an army that must be recruited by the AI.

Use script entities named with PlayerX_AttackTargetY to define positions that will be attacked by the army. Also you can use entities named with PlayerX_PatrolPointY to define positions were the army will patrol.

X shall be replaced with the player ID. Y shall be replaced with the index of the waypoint of players armies.

Parameters:

  • _ArmyName string Army identifier
  • _PlayerID number Id of player
  • _Strength number Strength of army
  • _Position string Army base position
  • _Area number Average action range
  • _TroopType string Army troop type

Defines an army of up to 6 different unit types that is spawned from a generator entity.

Use script entities named with PlayerX_AttackTargetY to define positions that will be attacked by the army. Also you can use entities named with PlayerX_PatrolPointY to define positions were the army will patrol.

X shall be replaced with the player ID. Y shall be replaced with the index of the waypoint of players armies.

Parameters:

  • _Strength number Strength of army
